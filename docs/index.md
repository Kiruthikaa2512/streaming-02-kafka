## Custom Project

### Dataset

The Kafka producer uses the `sales.csv` dataset located in the `data/` folder.

The dataset contains simulated retail sales transaction records including:
- order_id
- customer_id
- product_id
- quantity
- unit_price
- region_id
- payment_method
- device_type
- referral_source
- customer_note

The original sales dataset was used without modifying the source records.

### Kafka Messages

The producer sends retail sales transaction messages through Kafka using the topic:

`product-sales-case`

Each Kafka message contains retail order transaction details from the sales dataset.

The Kafka message key uses the `region_id` field to support regional streaming analysis.

### Consumer Processing

The custom consumer receives retail order transaction messages from Kafka and processes them one at a time.

The consumer:
- calculates order totals
- identifies high quantity fulfillment orders
- assigns fulfillment priority categories
- writes processed records into `consumed_sales.csv`

The custom consumer adds:
- `order_total`
- `fulfillment_priority`

Orders with quantity greater than or equal to 3 are flagged as:

`review-high-quantity-order`

All other records are categorized as:

`standard-order`
